Ship errors to IT with
one reusable modal

DevLogger gives every internal module the same polished error dialog. Users see a friendly message with “Mengerti” to dismiss and “Laporkan ke IT” to report — which saves the full error to MSSQL and emails your team automatically.

Quickstart

Three steps to wire it in

Recommended distribution: load the single hosted file and configure it once. No build step, no npm install.

Load the widget once per page

Point at the DevLogger host that serves this page. It injects its own styles — nothing else to import.

index.html 
<!-- Load once, anywhere before you use it -->
<script src="http://192.168.1.38/api/DevLogger/devlogger-modal.js"></script>

Configure it once at startup

Give it the API base URL and the logged-in user’s LMS token so reports can authenticate.

app.init.js 
DevLoggerModal.init({
  apiBaseUrl: "http://192.168.1.38/api/DevLogger/api/v1",
  token: window.LMS_TOKEN,      // the logged-in LMS token
  appName: "Nama Modul Anda",   // shown to IT in the report
  userId: currentUser.log_NIK   // optional, helps IT trace it
});

Show it whenever something fails

Pass a friendly message for the user and the raw error context for IT.

orders.js 
try {
  await saveOrder(payload);
} catch (err) {
  DevLoggerModal.showError({
    title: "Gagal Menyimpan Data",
    message: "Data belum tersimpan. Silakan coba lagi atau laporkan ke IT.",
    error: {
      url: "/api/orders",
      method: "POST",
      status_code: err.status,
      error_message: err.message,
      error_stack: err.stack,
      request_body: payload
    }
  });
}
That’s the whole integration. The user clicks Laporkan ke IT, the modal POSTs to /api/v1/errors/report, and DevLogger saves the row + emails the team. No per-app backend code.

Framework recipes

Drops into whatever you already run

React / Vite 
// once, e.g. in App.tsx
useEffect(() => {
  DevLoggerModal.init({
    apiBaseUrl: import.meta.env.VITE_DEVLOGGER + "/api/v1",
    token: getLmsToken(),
    appName: "Sales Dashboard"
  });
}, []);

// anywhere a call fails
catch (err) {
  DevLoggerModal.showError({
    message: "Gagal memuat laporan.",
    error: { url: "/api/reports", error_message: err.message }
  });
}
axios — global interceptor 
// catch every failed request in one place
axios.interceptors.response.use(null, (error) => {
  const res = error.response;
  DevLoggerModal.showError({
    message: "Permintaan ke server gagal.",
    error: {
      url: error.config?.url,
      method: (error.config?.method || "GET").toUpperCase(),
      status_code: res?.status,
      error_message: error.message,
      request_body: error.config?.data
    }
  });
  return Promise.reject(error);
});
Configuration

DevLoggerModal.init(options)

Set these once. Every later showError() inherits them.

Option Type Default Description
apiBaseUrl required to report string "" Base URL of the DevLogger API, e.g. https://host/api/v1.
token string "" LMS auth token; sent as both authentication and Authorization: Bearer.
getToken function null Called at report time to fetch a fresh token. Overrides token.
appName string "" Your module name — included in the notes IT receives.
userId string "" Current user’s NIK/id, attached to the report.
recipients string[] null Override the email recipients (to). Defaults to the server’s IT list.
saveToDb boolean true Persist the row to logging_mike as well as emailing.
autoContext boolean true Auto-fill url/method from the browser when omitted.
onReport function null Callback with the API response after a successful report.
onReportError function null Callback with the error if a report fails.

DevLoggerModal.showError(options)

Field Type Description
title string Modal heading. Default "Terjadi Kesalahan".
message string Friendly, user-facing explanation.
error object Technical context sent to IT — see below.
error.url string Endpoint or route that failed. required*
error.error_message string The raw error message. required*
error.method / status_code / error_stack — HTTP method, status, and stack trace.
error.request_body / headers / userid — Extra context saved with the row.
onClose function Called when the modal closes.

* If omitted, autoContext fills url/method from the browser and falls back to message for the error text.

Server API

The endpoint behind the button

The modal’s report button calls this. You can also call it directly from any backend.

POST  /api/v1/errors/report

Saves the error to logging_mike (when save_to_db) and emails the IT recipients. Auth required · rate-limited 10 req / 15 min per IP.

request.json 
{
  "method": "POST",
  "url": "/api/orders",
  "status_code": 500,
  "error_message": "Timeout saving order",
  "error_stack": "Error: Timeout...\n  at save()",
  "request_body": { "orderId": 42 },
  "userid": "12345",
  "notes": "App: Sales Dashboard",
  "save_to_db": true
}
200 response.json 
{
  "success": true,
  "message": "Error reported and alert sent",
  "data": {
    "messageId": "<...@lapilabs>",
    "recipients": ["ITnotifLMS@lapilabs.co.id"],
    "saved_to_db": true,
    "id": 1284
  }
}
Headers. Send the LMS token as authentication: <token> or Authorization: Bearer <token>. The widget sets both for you when you provide token.
